# IT Support Troubleshooting Home Lab

## Project Overview

This project demonstrates a simulated IT support troubleshooting environment designed to replicate common issues handled by help desk technicians. The lab was created using a Windows virtual machine to practice diagnosing and resolving system, network, and user account related problems.

The objective of the lab was to build practical troubleshooting experience similar to real-world IT support scenarios.

---

## Technologies Used

- Windows Virtual Machine
- Command Prompt
- Windows Network Settings
- System Diagnostics Tools
- Task Manager
- System Information

---

## Lab Environment

A Windows virtual machine was used to simulate an IT support workstation environment. Several system and network issues were intentionally created in order to practice identifying the root cause and applying the appropriate fix.

The troubleshooting process followed typical IT support workflow:

1. Identify the issue  
2. Diagnose the cause  
3. Apply a solution  
4. Verify that the issue is resolved  

---

## Troubleshooting Scenarios

### 1. Network Connectivity Issue

**Problem**

The system was unable to connect to the network. Basic internet connectivity was unavailable and network resources could not be accessed.

**Diagnosis**

Network diagnostics were performed using:

- `ipconfig`
- Network adapter settings
- Ping tests

The system showed incorrect or missing network configuration.

**Solution**

The network configuration was corrected and the connection was re-established. After the fix, successful ping tests confirmed connectivity.

---

### 2. System Diagnostics Investigation

**Problem**

The system performance needed to be analyzed to understand how resources were being used.

**Diagnosis**

System diagnostic tools were used to review system performance and configuration.

Tools used:

- Task Manager
- System Information
- Command Prompt

These tools helped identify running processes and system resource usage.

---

### 3. User Account Management

**Problem**

User account access and configuration needed to be reviewed to simulate a typical help desk support request.

**Diagnosis**

User account settings and permissions were examined through Windows account management tools.

**Solution**

The user account configuration was verified and updated to ensure proper access.

This lab successfully simulated common IT support scenarios including network troubleshooting, system diagnostics, and user account investigation. The project helped build hands-on experience with diagnosing and resolving technical issues in a controlled environment.
